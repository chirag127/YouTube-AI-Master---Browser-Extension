Privacy Policy

TL;DR (Summary)

  • All data stays on your device (local storage only)

  • Only one permission requested: storage
  • Open source - verify the code yourself

1. Data Collection & Storage

1.1 What Data We Store Locally

The Extension stores the following data exclusively on your device using Chrome's local storage:

Video Analysis Data

  • Video ID (YouTube's public identifier)
  • Video metadata (title, author, duration, view count)
  • Generated AI summaries and analysis
  • Extracted transcripts
  • Segment classifications (intro, sponsor, outro, etc.)
  • Chat conversation history

User Preferences

  • API keys (Gemini, TMDB, IGDB, NewsData, etc.)
  • Selected AI model (Gemini Flash, Pro, etc.)
  • Output language preference
  • Summary length preference
  • Auto-analysis settings
  • Auto-skip segment preferences
Purpose: This data enables the Extension to provide instant analysis for previously watched videos (caching), remember your preferences across sessions, maintain chat conversation context, and function offline for cached videos.
Retention: Data is stored indefinitely until you manually clear it via the Extension settings, uninstall the Extension, or clear Chrome's local storage.

2. Permissions Explained

The Extension requests only ONE permission from Chrome:

storage Permission

Why We Need It:

  • Save API Keys Securely: Your Gemini API key and optional service keys are stored in Chrome's encrypted sync storage, allowing seamless access across your devices while maintaining security.
  • Cache Video Analysis: Previously analyzed videos are cached locally, enabling instant retrieval without redundant API calls, saving time and API quota.
  • Store User Preferences: Your settings (output language, AI model selection, summary length, auto-skip rules) are persisted across sessions.
  • Maintain Chat History: Conversation context is stored locally to enable multi-turn dialogues with the AI about video content.

What It Does NOT Allow:

  • ❌ Access to your browsing history
  • ❌ Access to data from other websites
  • ❌ Access to your personal information (name, email, etc.)
  • ❌ Tracking your activity across the web
  • ❌ Reading or modifying data on non-YouTube websites

Technical Implementation:

  • Uses chrome.storage.local for video cache and analysis data (device-specific)
  • Uses chrome.storage.sync for API keys and preferences (synced across Chrome browsers)
  • All storage operations are scoped to the extension's isolated storage area
  • No access to cookies, passwords, or other browser data

Chrome's Documentation: chrome.storage API

Permissions We Do NOT Request

activeTab

We do not request access to your active tab. The Extension only interacts with YouTube pages via content scripts that run exclusively on youtube.com. We cannot access your active tab's content outside of YouTube.

tabs

We cannot access your tabs list, tab URLs, or switch between tabs programmatically.

history

We cannot access your browsing history or track which websites you visit.

cookies

We cannot read or modify cookies from any website, including YouTube.

webRequest

We do not intercept or modify network requests globally. Transcript extraction on YouTube pages uses content script injection, not network interception.

scripting

Removed in v2.0.0. Previously requested but unused. The Extension uses declarative content scripts instead of programmatic injection.

3. Host Permissions Explained

The Extension declares host_permissions for specific domains to enable API communication:

YouTube Domains

https://www.youtube.com/*, https://youtube.com/*

  • Purpose: Inject content scripts to display the AI widget, timeline markers, and segment controls
  • Scope: Only YouTube video pages
  • Data Access: Video ID, title, author, transcript (publicly available data)

Google Gemini AI

https://generativelanguage.googleapis.com/*

  • Purpose: Send video transcripts and metadata to Gemini AI for analysis
  • Data Sent: Video transcript, title, description, your custom prompt
  • Required: Yes (core functionality)

Invidious Instances

Multiple domains (e.g., https://*.invidious.io/*)

  • Purpose: Fetch video transcripts when YouTube's direct method fails
  • Data Sent: Video ID only
  • Required: No (fallback method)

Piped Instances

Multiple domains (e.g., https://pipedapi.kavin.rocks/*)

  • Purpose: Fetch video transcripts (privacy-friendly alternative)
  • Data Sent: Video ID only
  • Required: No (fallback method)

Wildcard Permission

*://*/*

  • Purpose: Enable communication with user-configured optional API services (TMDB, IGDB, NewsData, etc.)
  • Scope: Only used when you explicitly configure API keys for optional services
  • Data Sent: Video title and metadata to configured services only
  • Required: No (optional enhancements)
Important: The wildcard permission does NOT grant access to all websites. It only allows the Extension to make HTTP requests to external APIs when you explicitly configure them. The Extension cannot read or modify content on arbitrary websites.

4. Data Transmission

4.1 What Data Leaves Your Browser

The Extension transmits data to external services only when you explicitly trigger an analysis:

Service Data Sent Purpose Required
Google Gemini AI Video transcript, metadata, chat messages Generate AI-powered summaries and answers ✅ Yes
TMDB Video title Movie & TV show metadata ⚪ Optional
IGDB Video title Video game metadata ⚪ Optional
NewsData.io Video title, keywords News article search ⚪ Optional

SponsorBlock Video ID Community-verified segments ⚪ Optional

4.2 What Data NEVER Leaves Your Browser

  • ❌ Your browsing history
  • ❌ Your personal information (name, email, etc.)
  • ❌ Your Chrome profile data
  • ❌ Data from other websites
  • ❌ Your YouTube watch history (beyond the current video)

5. Security Measures

HTTPS Only: All external API requests use HTTPS encryption.
No Third-Party Tracking: The Extension does not integrate any analytics, telemetry, or tracking services (Google Analytics, Sentry, etc.).
API Key Protection:
  • API keys are stored in Chrome's secure storage
  • Never logged to console (in production builds)
  • Never transmitted except to the official API endpoints
  • Never shared with third parties

6. Your Rights & Controls

6.1 Data Access

You can access all data stored by the Extension:

  1. Open Chrome DevTools (F12)
  2. Go to Application tab
  3. Expand Local Storage → chrome-extension://[extension-id]
  4. View all stored data

6.2 Data Deletion

Clear Specific Video Data:

  • Open Extension settings → History
  • Click "Delete" next to any video

Clear All Data:

  • Open Extension settings → General
  • Click "Clear All History"

Uninstall Extension:

  • Go to chrome://extensions/
  • Click "Remove" on YouTube AI Navigator
  • All local data is automatically deleted
